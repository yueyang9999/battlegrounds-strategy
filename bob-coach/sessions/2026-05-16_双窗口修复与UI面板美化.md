# 2026-05-16 双窗口修复与UI面板美化

## 问题诊断

本次会话主要解决覆盖层插件在2060分辨率(2560x1600)下的两大问题：Control窗口交互失效、决策引擎无数据。

## 已修复

### 双窗口模式
- **additionalArguments 注入**: `createWindows()` 中 displayWin/controlWin 分别注入 `--bob-mode=display` 和 `--bob-mode=control`，preload.js 通过 `process.argv` 读取
- **指针事件**: `body { pointer-events: none }` 对 Display 穿透，`body.mode-control { pointer-events: auto }` 对 Control 恢复
- **智能显隐**: 游戏检测到 → `setIgnoreMouseEvents(false)` + `setOpacity(0.7)` 可交互；未检测到 → `setIgnoreMouseEvents(true, {forward: true})` 透明穿透
- **setParentWindow 回退**: Windows 透明窗口父子兼容问题 → 独立窗口 + `moveTop()`

### 主进程稳定性
- **globalShortcut 报错**: 4处 `register/unregister/unregisterAll` 加 `app.isReady()` 守卫
- **log.config 修复**: `ensureHearthstoneLogging()` 写入标准格式 `[Power] LogLevel=1 FilePrinting=true`，自动创建 Logs 目录
- **日志源校验**: `findGameLogPaths()` 只纳入含游戏事件(PowerTaskList/TAG_CHANGE/FULL_ENTITY)的日志，排除 HDT 纯诊断日志

### 决策引擎链路
- **game-state-update handler** 新增决策引擎调用（1s 节流）: `computeCombatPrediction()` + `runDecisionEngine()` + `renderAll()`
- Demo 模式检测到真实对局自动隐藏
- 无场面匹配时 `getFallbackComps()` 基于种族+英雄提供兜底流派推荐

### 拔线按钮
- `netsh` 需要管理员权限 → 视觉反馈始终生效，防火墙阻塞可选

### UI 面板美化
- **comp-panel**: `height: 72vh` → `max-height: 68vh; height: auto` 面板随内容自适应
- **全局滚动条**: 4px 半透明，`::-webkit-scrollbar` 统一风格
- **紧凑布局**: padding/gap/margin 全面缩减 20-30%
- **空区域折叠**: `.comp-section.no-content { display: none }`，JS 在无数据时自动添加

## 保留展开

- Power.log 仍为生成：Logs 目录已创建、log.config 已修复，用户需重启炉石。可能需排查炉石版本日志路径差异
- 流派推荐数据依赖 Power.log 实际游戏数据流入后验证

- `main.js` — additionalArguments、窗口追踪、日志配置修复、shortcut 守卫、拔线视觉反馈
